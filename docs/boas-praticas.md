# Boas Práticas de Git

Este arquivo descreve boas práticas no uso de Git e GitHub.

---

## 1. Branches claras
- Crie branches com nomes descritivos e padronizados.
- Exemplo:
  - `feature/adicionar-menu`
  - `hotfix/corrigir-login`

---

## 2. Commits pequenos
- Faça commits pequenos e focados em uma única alteração.
- Evite commits grandes com muitas mudanças diferentes.

---

## 3. Pull Requests curtos
- Mantenha os Pull Requests pequenos e fáceis de revisar.
- Cada PR deve conter **uma funcionalidade ou correção específica**.

---

## 4. Mensagens descritivas
- Use mensagens de commit claras e semânticas:
  - `feat: adicionar barra de navegação`
  - `fix: corrigir bug no login`
  - `docs: atualizar README`

---

## 5. Releases organizadas
- Utilize tags e Semantic Versioning para organizar releases:
  - `v1.0.0` – primeira versão estável
  - `v1.1.0` – novas funcionalidades
  - `v1.1.1` – correções rápidas (hotfix)
